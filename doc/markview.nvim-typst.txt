------------------------------------------------------------------------------
enable

  ‚Ä¢ type: `boolean`
    default: `true`

Allows viewing typst in preview.

------------------------------------------------------------------------------
code_blocks

  ‚Ä¢ type: `markview.config.typst.code_blocks`[^3]
    default[^4]

Changes how code blocks look.

>lua
    ---@type markview.config.typst.code_blocks
    code_blocks = {
        enable = true,

        hl = "MarkviewCode",

        min_width = 60,
        pad_amount = 3,
        pad_char = " ",

        style = "block",

        text = "Û∞£ñ Code",
        text_direction = "right",
        text_hl = "MarkviewIcon5"
    },
<

enable ~

  ‚Ä¢ type: `boolean`
    default: `true`

Self-explanatory.

hl ~

  ‚Ä¢ type: `string`
    default: `"MarkviewCode"`

Highlight group used for the background.

min_width ~

  ‚Ä¢ type: `integer`
    default: `60`

Minimum width of code blocks.

pad_amount ~

  ‚Ä¢ type: `integer`
    default: `2`

Width of the left & right border/padding.

pad_char ~

  ‚Ä¢ type: `string`
    default: `" "`

Character used for the borders/paddings.
sign
  ‚Ä¢ type: `string`

Text to show in the signcolumn.

sign_hl ~

  ‚Ä¢ type: `string`

Highlight group used for the `sign`.

style ~

  ‚Ä¢ type: `"simple" | "block"` default: `"block"`

Changes how code blocks are shown. Supported values are,

  ‚Ä¢ `"simple"` Entire line is highlighted. Useful when `wrap` is enabled.
  ‚Ä¢ `"block"` A block is created around the code block and paddings are added
    before & after each line.
text

  ‚Ä¢ type: `string`
    default: `"Û∞£ñ Code"`

Text to show on the top-left/top-right side of the code block.

text_direction ~

  ‚Ä¢ type: `"left" | "right"`
    default: `"right"`

Changes which side `text` should be shown on.

text_hl ~

  ‚Ä¢ type: `string`

Highlight group used for `text`.

------------------------------------------------------------------------------
code_spans

  ‚Ä¢ type: `markview.config.typst.code_spans`[^8]
    default[^9]

Changes how code spans look. See how inline elements are configured[^10].

------------------------------------------------------------------------------
escapes

  ‚Ä¢ type: `{ enable: boolean }`
    default: `{ enable = true }`

Allows showing the escaped characters in previews.

  ‚Ä¢ type: `boolean`
    default: `true`

Self-explanatory.

enable ~

  ‚Ä¢ type: `boolean`
    default: `true`

Self-explanatory.

shift_width ~

  ‚Ä¢ type: `integer`
    default: `1`

Number of spaces to add before the heading per level.

heading_[n] ~

  ‚Ä¢ type: `markview.config.typst.headings.typst`[^13]

Configuration for level `n` headings.

HL

  ‚Ä¢ type: `string`

Highlight group for the heading.
icon

‚ñå This has no effect if `style` is set to `simple`.

  ‚Ä¢ type: `string`

ICON_HL

‚ñå üìú Note
‚ñå This has no effect if `style` is set to `simple`.

  ‚Ä¢ type: `string`

Highlight group for icon[^16].
sign

‚ñå üìú Note
‚ñå This has no effect if `style` is set to `simple`.

  ‚Ä¢ type: `string`

Text to show in the signcolumn.

SIGN_HL

‚ñå üìú Note
‚ñå This has no effect if `style` is set to `simple`.

  ‚Ä¢ type: `string`

Highlight group for `sign`.

STYLE

  ‚Ä¢ type: `"simple" | "icon"`

Heading preview style. Possible values are,

  ‚Ä¢ `"simple"`,
    Only Highlight the line.

  ‚Ä¢ `"icon"`,
    Icons & signs with highlight.
